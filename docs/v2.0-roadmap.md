## 二、SEO 基础设施（P0 — 第 1-2 周）

### 2.1 Schema.org 结构化数据
**为什么最重要：** 没有 Schema 的博客在 Google 搜索结果中缺少富片段（rich snippets），点击率低 20-30%。

| 页面类型 | Schema 类型 | 关键字段 |
|----------|------------|----------|
| 首页 | `Organization` + `WebSite` | name, url, logo, sameAs |
| 文章页 | `Article` + `BlogPosting` | headline, author, datePublished, dateModified, image, wordCount |
| 关于页 | `AboutPage` | name, description |
| 博客列表 | `CollectionPage` | name, description |

实现方式：在 `BlogPost.astro` 布局中已有 JSON-LD（`BlogPosting`），需补全：
- 首页加 `Organization` + `WebSite`（含 `potentialAction: SearchAction`）
- `wordCount` 字段（从 MDX 内容计算）
- `author` 改为具体作者名（目前是 "HardStack"）

### 2.2 Title / Description 优化
当前问题及修复：

| 页面 | 问题 | 修复 |
|------|------|------|
| 首页 | Title "HardStack — HardStack" 重复 | → "HardStack — 深度开源项目安装教程与踩坑记录" |
| 404 | Title 过短 | → "页面未找到 — HardStack" |
| Coolify EN | Description 91 字符偏短 | 补充到 120-155 字符 |
| DBX EN | Title 64 字符过长 | 缩短到 60 以内 |

### 2.3 Google Search Console 接入
- 在 `BaseHead.astro` 加 `<meta name="google-site-verification" content="...">`
- 提交 sitemap 到 GSC
- 监控索引覆盖率和搜索表现

### 2.4 robots.txt 完善
当前 `robots.txt` 缺失。需添加：
```
User-agent: *
Allow: /
Sitemap: https://hardstack.org/sitemap-index.xml
```

## 四、功能升级（P1-P2 — 第 3-8 周）

### 4.1 阅读体验增强

**a) 阅读进度条**
- 在文章顶部加一条细进度条，随滚动填充
- 实现：CSS `position: fixed` + JS `scroll` 事件
- 颜色跟随当前 template 的 accent 色

**b) 字数统计 + 预计阅读时间**
- 在文章头部 meta 区域显示
- 已有 `timeToRead` 字段，需确保前端正确显示
- 补充字数统计（中文按字符，英文按单词）

**c) 目录（TOC）**
- 从 H2/H3 自动生成侧边目录
- 滚动高亮当前章节
- 移动端折叠为下拉菜单

### 4.2 邮件订阅 / Newsletter

**方案选择：**
| 方案 | 成本 | 复杂度 | 推荐 |
|------|------|--------|------|
| Buttondown | 免费 100 订阅 | 低 | ✅ 推荐 |
| Resend + 自建 | 免费 3000 封/月 | 中 | 备选 |
| Substack | 免费 | 低 | 不推荐（品牌外泄） |

**实现：**
- 文章底部加订阅表单
- 首页 hero 区加 CTA
- 新文章发布时自动通知订阅者

### 4.3 相关文章推荐

**方案：** 基于标签匹配
- 每篇文章底部显示 2-3 篇相关文章
- 匹配规则：同 category > 同 tags > 同 difficulty
- 优先推荐同语言文章，其次推荐另一语言版本

实现：在 `BlogPost.astro` 布局中加一个 slot，由页面传入相关文章列表。

### 4.4 读者互动（点赞 / 评分）

**方案：** 使用 Cloudflare D1（已有基础设施）
- 每篇文章底部加 👍 按钮
- 数据存 D1（复用现有 `/api/views` 架构）
- 显示点赞数，不显示谁点的（匿名）
- 防刷：每篇文章每 IP 每天限 1 次

## 六、页面视觉效果优化（P1-P2 — 第 3-10 周）

### 6.1 文章页视觉增强

**a) 代码块升级**
- 行号显示（可选开关）
- 代码块右上角加「复制」按钮，一键复制代码
- 语言标签（如 `bash`、`yaml`）显示在代码块左上角
- 长代码块折叠（超过 30 行自动折叠，点击展开）

**b) 图片体验**
- 图片点击放大（lightbox），支持左右切换
- 图片加载占位符（blurhash 或骨架屏）
- 图片 caption 自动居中，支持 markdown 图片标注语法

**c) 提示框 / Callout 组件**
- 支持 `tip`、`warning`、`danger`、`info` 四种样式
- 在 MDX 中用 blockquote 语法：`> [!TIP]` / `> [!WARNING]`
- 颜色跟随当前 template 的 accent 色

### 6.2 首页视觉升级

**a) Hero 区域动效**
- Terminal 统计卡片入场动画（fade-in + 轻微上浮）
- 文章列表 stagger 动画（已有，可增强时长和缓动曲线）
- Hero CTA 按钮 hover 光泽效果

**b) 文章卡片增强**
- 卡片 hover 时轻微上浮 + 阴影加深
- 分类 pill 加渐变背景（而非纯色）
- 阅读时间加 icon 前缀

### 6.3 全局视觉统一

**a) 过渡动画规范**
- 所有交互元素统一 `transition: all 0.2s cubic-bezier(0.22, 1, 0.36, 1)`
- 页面切换fade 效果（Astro view transitions API）
- `prefers-reduced-motion` 全面支持

**b) 间距系统**
- 统一组间距 token：`--space-xs` / `--space-sm` / `--space-md` / `--space-lg` / `--space-xl`
- 替换当前硬编码的 `rem` / `px` 值

**c) 滚动指示器**
- 文章页右侧加迷你 TOC 进度条（细竖线，当前章节高亮）
- 首页向下滚动时 header 底部加微妙阴影变化

### 6.4 移动端视觉优化

- 底部导航栏（可选）：固定在屏幕底部的快速导航
- 触摸友好的交互区域（按钮最小 44x44px）
- 移动端代码块横向滚动优化（加渐变遮罩提示可滚动）
