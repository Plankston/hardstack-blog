### Task 3: Add WebSite + Organization schema to Homepage

**Files:**
- Modify: `src/components/Homepage.astro`

**Problem:** Homepage has zero Schema.org structured data. Needs `Organization` + `WebSite` with `potentialAction: SearchAction` for Google sitelinks search box.

**Interfaces:**
- Consumes: site URL (`https://hardstack.org`), site name
- Produces: JSON-LD script tag in `<head>`

- [ ] **Step 1: Add JSON-LD to Homepage.astro**

In `src/components/Homepage.astro`, inside the `<head>` section (after `BaseHead`), add:

```astro
<script is:inline type="application/ld+json" set:html={JSON.stringify({
  '@context': 'https://schema.org',
  '@type': 'Organization',
  name: 'HardStack',
  url: 'https://hardstack.org',
  logo: 'https://hardstack.org/favicon.svg',
  sameAs: ['https://github.com/Plankston/hardstack-blog'],
})} />
<script is:inline type="application/ld+json" set:html={JSON.stringify({
  '@context': 'https://schema.org',
  '@type': 'WebSite',
  name: 'HardStack',
  url: 'https://hardstack.org',
  potentialAction: {
    '@type': 'SearchAction',
    target: {
      '@type': 'EntryPoint',
      urlTemplate: 'https://hardstack.org/blog/?q={search_term_string}',
    },
    'query-input': 'required name=search_term_string',
  },
})} />
```

- [ ] **Step 2: Verify build passes**

Run: `npm run build`
Expected: Build succeeds.

- [ ] **Step 3: Commit**

```bash

git commit -m "feat: add Organization + WebSite JSON-LD to homepage"
```

### Task 4: Enhance Article schema in BlogPost.astro

**Files:**
- Modify: `src/layouts/BlogPost.astro`

**Problem:** Existing Article schema is missing `image`, `wordCount`, `timeRequired`. `keywords` is always empty.

**Interfaces:**
- Consumes: post frontmatter (title, description, pubDate, updatedDate, category, difficulty, timeToRead, heroImage)
- Produces: enhanced JSON-LD

- [ ] **Step 1: Enhance the JSON-LD object in BlogPost.astro**

Replace the existing `jsonLd` object (lines 37-60) with:

```typescript
const wordCount = post?.body ? post.body.split(/\s+/).length : undefined;

const jsonLd = {
  '@context': 'https://schema.org',
  '@type': 'BlogPosting',
  headline: title,
  description: description,
  datePublished: pubDate.toISOString(),
  dateModified: updatedDate ? updatedDate.toISOString() : pubDate.toISOString(),
  author: {
    '@type': 'Person',
    name: 'Plankston',
    url: 'https://github.com/Plankston',
  },
  publisher: {
    '@type': 'Organization',
    name: 'HardStack',
    url: 'https://hardstack.org',
    logo: {
      '@type': 'ImageObject',
      url: 'https://hardstack.org/favicon.svg',
    },
  },
  mainEntityOfPage: {
    '@type': 'WebPage',
    '@id': `https://hardstack.org${Astro.url.pathname}`,
  },
  inLanguage: locale === 'zh-CN' ? 'zh-CN' : 'en',
  keywords: category ? [category] : [],
  wordCount: wordCount,
  timeRequired: timeToRead ? `PT${timeToRead}M` : undefined,
  image: slug ? `https://hardstack.org/og/${slug}.png` : undefined,
};
```

Note: This requires accessing `post.body` which is available in the layout via `Astro.props`. We need to check if `post` is available as a prop. If not, we can skip `wordCount` for now (ponytail: skip wordCount if body not available).

```astro

  '@type': 'BreadcrumbList',

      '@type': 'ListItem',
      position: 1,
      name: 'Home',
      item: locale === 'zh-CN' ? 'https://hardstack.org/zh-CN/' : 'https://hardstack.org/',

      '@type': 'ListItem',
      position: 2,
      name: 'Blog',
      item: locale === 'zh-CN' ? 'https://hardstack.org/zh-CN/blog/' : 'https://hardstack.org/blog/',

      '@type': 'ListItem',
      position: 3,
```
