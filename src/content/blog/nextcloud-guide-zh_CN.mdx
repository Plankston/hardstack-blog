## 为什么要自托管云盘？

三个原因，按重要性排序：

1. **数据主权（数据主权）**。你的文件被云端 AI 扫描——用于索引、用于广告、用于你根本没同意的用途。自托管意味着唯一读取你文件的实体就是你自己。

2. **成本**。Google One 2TB = $100/年。Dropbox Plus 2TB = $120/年。五年下来就是 $600-$1,200。一块 $150 的硬盘能跑一辈子。Nextcloud 免费。

3. **没有厂商锁定（无厂商锁定）**。Dropbox 砍了无限计划。Google 砍了 Google Photos 无限存储。等他们砍下一个的时候，你的数据一起没了——除非它已经在你自己的服务器上。

## Docker 部署

### 第一步：创建项目目录

```bash
mkdir -p /opt/nextcloud
cd /opt/nextcloud
```

### 第二步：创建 docker-compose.yml

```yaml
services:
  db:
    image: mariadb:10.11
    container_name: nextcloud-db
    restart: unless-stopped
    command: --transaction-isolation=READ-COMMITTED --log-bin=binlog --binlog-format=ROW
    volumes:
      - db_data:/var/lib/mysql
    environment:
      - MYSQL_ROOT_PASSWORD=your-root-password
      - MYSQL_DATABASE=nextcloud
      - MYSQL_USER=nextcloud
      - MYSQL_PASSWORD=your-nextcloud-password

  redis:
    image: redis:7-alpine
    container_name: nextcloud-redis
    restart: unless-stopped

  app:
    image: nextcloud:stable
    container_name: nextcloud-app
    restart: unless-stopped
    depends_on:
      - db
      - redis
    volumes:
      - nextcloud_data:/var/www/html
    environment:
      - MYSQL_HOST=db
      - MYSQL_DATABASE=nextcloud
      - MYSQL_USER=nextcloud
      - MYSQL_PASSWORD=your-nextcloud-password
      - REDIS_HOST=redis
      - NEXTCLOUD_TRUSTED_DOMAINS=yourdomain.com
      - OVERWRITEPROTOCOL=https

  collabora:
    image: collabora/code
    container_name: nextcloud-collabora
    restart: unless-stopped
    ports:
      - 127.0.0.1:9980:9980
    environment:
      - domain=yourdomain\.com
      - extra_params=--o:ssl.enable=false --o:ssl.termination=true

volumes:
  db_data:
  nextcloud_data:
```

### 第三步：启动服务

```bash
docker compose up -d
```

### 第四步：验证部署

```bash
# 检查所有容器是否正常运行
docker compose ps

# 查看 Nextcloud 日志
docker compose logs app --tail 50

# 测试 HTTP 响应
curl -I http://localhost
# 应该返回 200 或 301
```

**就这样。** 20 分钟，一个完整的云存储平台。

**注意：** `OVERWRITEPROTOCOL=https` 这一行在你用了反向代理 + SSL 的情况下至关重要。没有它，Nextcloud 会生成混合内容 URL，你的移动端 App 会默默地挂掉。

## 初始配置

### 访问 Web 界面

1. 在浏览器打开 `https://yourdomain.com`
2. 创建管理员账户（别用 "admin" 作用户名）
3. 选数据库——要性能选 **PostgreSQL**，要兼容性选 **MariaDB**

### 必调设置

进入 **管理设置 → 概览**，把所有警告都修了。Nextcloud 对安全是偏执狂——这是特性，不是 bug。

**关键设置：**

- **管理 → 概览**：运行安全检查，修掉所有问题
- **管理 → 分享**：如果你想保护隐私，禁用"允许用户通过链接分享"
- **管理 → 日志**：日志级别设成 Warning（Info 太吵了）
- **个人 → 安全**：开启双因素认证（2FA）

### 安装推荐应用

```bash
# 通过 CLI 安装（比 Web 界面快）
docker compose exec app php occ app:install files_external
docker compose exec app php occ app:install calendar
docker compose exec app php occ app:install contacts
docker compose exec app php occ app:install deck
docker compose exec app php occ app:install notes
```

## WebDAV 访问

WebDAV（Web 分布式创作和版本控制）是 Nextcloud 的杀手锏。它能让你在任何操作系统上把 Nextcloud 挂载为网络驱动器。

### Linux (Nautilus/Files)

1. 打开 Files
2. 点击 "Other Locations"
3. 输入：`davs://yourdomain.com/remote.php/dav/files/username/`
4. 输入凭据

### macOS (Finder)

1. 打开 Finder
2. 按 `Cmd+K`
3. 输入：`https://yourdomain.com/remote.php/dav/files/username/`
4. 输入凭据

### Windows (File Explorer)

1. 右键点击 "此电脑"
2. 点击 "映射网络驱动器"
3. 输入：`https://yourdomain.com/remote.php/dav/files/username/`
4. 勾选 "使用其他凭据连接"
5. 输入凭据

### 验证 WebDAV

```bash
# 测试 WebDAV 端点
curl -u username:password https://yourdomain.com/remote.php/dav/files/username/
# 应该返回文件列表的 XML

# 上传测试文件
echo "hello" | curl -T - -u username:password https://yourdomain.com/remote.php/dav/files/username/test.txt

# 再下载回来
curl -u username:password https://yourdomain.com/remote.php/dav/files/username/test.txt
# 应该返回 "hello"
```

## Collabora 在线协作

[Collabora Online](https://www.collaboraoffice.com/)（在线协作）把 Nextcloud 变成一个完整的 Office 套件——Word、Excel、PowerPoint 的等价物，全部自托管。

### 配置

Collabora 容器已经在你的 `docker-compose.yml` 里了。现在把它连到 Nextcloud：

1. 进入 **应用 → 已禁用的应用**
2. 找到 **Collabora Online** 并启用
3. 进入 **管理 → Collabora Online**
4. 设置 WOPI URL：`http://collabora:9980`（Docker 内部网络）
5. 保存

### 测试

1. 上传一个 `.docx`、`.xlsx` 或 `.pptx` 文件到 Nextcloud
2. 点击文件——它会在 Collabora 中打开
3. 编辑、分享、协作——全部在浏览器里完成

### 联邦协作（联邦协作）

Nextcloud 支持联邦分享——和其他 Nextcloud 实例的用户分享文件：

1. 进入 **设置 → 分享**
2. 启用"允许用户通过联邦云分享"
3. 分享给 `user@other-nextcloud-domain.com`

## 性能调优

### 启用 Redis 缓存

已经在 `docker-compose.yml` 里配好了。验证一下是否生效：

```bash
docker compose exec app php occ config:set memcache.local '\OC\Memcache\APCu'
docker compose exec app php occ config:set memcache.distributed '\OC\Memcache\Redis'
docker compose exec app php occ config:set memcache.locking '\OC\Memcache\Redis'
```

### 提升 PHP 内存限制

```bash
# 创建自定义 PHP 配置
cat > custom.ini << 'EOF'
upload_max_filesize = 16G
post_max_size = 16G
memory_limit = 1G
max_execution_time = 3600
max_input_time = 3600
EOF

# 复制到容器
docker cp custom.ini nextcloud-app:/usr/local/etc/php/conf.d/custom.ini
docker compose restart app
```

### 启用 HTTP/2

如果用 Nginx 做反向代理：

```nginx
server {
    listen 443 ssl http2;
    # ... 其余配置
}
```

### 监控性能

```bash
# 检查 Nextcloud 状态
docker compose exec app php occ status

# 运行完整性检查
docker compose exec app php occ integrity:check-core

# 检查后台任务
docker compose exec app php occ background:cron
```

## 备份策略

### 备份什么

1. **数据库**——所有元数据、分享、版本
2. **Nextcloud 数据目录**——实际文件
3. **配置**——`config/config.php`

### 自动备份脚本

```bash
#!/bin/bash
# /opt/nextcloud/backup.sh

BACKUP_DIR="/opt/backups/nextcloud"
DATE=$(date +%Y%m%d_%H%M%S)

mkdir -p $BACKUP_DIR

# 开启维护模式
docker compose exec -T app php occ maintenance:mode --on

# 备份数据库
docker compose exec -T db mysqldump -u root -p'your-root-password' nextcloud | gzip > $BACKUP_DIR/db_$DATE.sql.gz

# 备份数据目录
tar -czf $BACKUP_DIR/data_$DATE.tar.gz -C /var/lib/docker/volumes/nextcloud_nextcloud_data/_data .

# 备份配置
tar -czf $BACKUP_DIR/config_$DATE.tar.gz -C /var/lib/docker/volumes/nextcloud_nextcloud_data/_data/config config.php

# 关闭维护模式
docker compose exec -T app php occ maintenance:mode --off

# 只保留最近 7 个备份
ls -t $BACKUP_DIR/db_*.sql.gz | tail -n +8 | xargs rm -f
ls -t $BACKUP_DIR/data_*.tar.gz | tail -n +8 | xargs rm -f
ls -t $BACKUP_DIR/config_*.tar.gz | tail -n +8 | xargs rm -f

echo "Backup completed: $DATE"
```

### 用 Cron 定时执行

```bash
# 添加到 crontab
crontab -e

# 每天凌晨 2 点执行
0 2 * * * /opt/nextcloud/backup.sh >> /var/log/nextcloud-backup.log 2>&1
```

### 从备份恢复

```bash
# 停止 Nextcloud
docker compose down

# 恢复数据库
gunzip < /opt/backups/nextcloud/db_YYYYMMDD_HHMMSS.sql.gz | docker compose exec -T db mysql -u root -p'your-root-password' nextcloud

# 恢复数据
tar -xzf /opt/backups/nextcloud/data_YYYYMMDD_HHMMSS.tar.gz -C /var/lib/docker/volumes/nextcloud_nextcloud_data/_data/

# 恢复配置
tar -xzf /opt/backups/nextcloud/config_YYYYMMDD_HHMMSS.tar.gz -C /var/lib/docker/volumes/nextcloud_nextcloud_data/_data/config

# 启动 Nextcloud
docker compose up -d

# 修复权限
docker compose exec app chown -R www-data:www-data /var/www/html
```

## 常见坑与解决方案

### 移动端报 "Can't connect to the server"

**原因：** 缺少 `NEXTCLOUD_TRUSTED_DOMAINS` 或 `OVERWRITEPROTOCOL`。

**修复：** 把你的域名加到受信任域名，并设置协议：

```yaml
environment:
  - NEXTCLOUD_TRUSTED_DOMAINS=yourdomain.com
  - OVERWRITEPROTOCOL=https
```

### 大文件上传失败

**原因：** PHP 限制或反向代理超时。

**修复：** 提升 PHP 限制（见性能调优部分），并在 Nginx 配置中添加：

```nginx
client_max_body_size 16G;
proxy_read_timeout 3600s;
```

### Collabora 显示 "Failed to connect"

**原因：** Collabora 容器连不上 Nextcloud。

**修复：** 确保 Collabora 的 `domain` 和你的实际域名匹配（转义点号）：

```yaml
environment:
  - domain=yourdomain\.com
```

### 文件同步慢

**原因：** 缺少 Redis 或分块设置不对。

**修复：** 启用 Redis 缓存并增大分块大小：

```bash
docker compose exec app php occ config:system:set chunked_file_size --value=100 --type=integer
```

## FAQ

### Nextcloud vs Seafile vs ownCloud？

| 使用场景 | 推荐 |
|----------|---------------|
| **全能生产力平台** | **Nextcloud**——文件同步 + Office 套件 + 日历 + 通讯录 + 300+ 应用 |
| **只要快速可靠的文件同步** | **Seafile**——块级同步，资源占用低 |
| **遗留企业需求** | **ownCloud**——从 Nextcloud fork，企业导向 |

### 我需要多大存储？

取决于你的使用场景：
- **仅个人文件：** 500GB-1TB 足够
- **家庭照片/视频：** 2-4TB
- **完全替代 Google Drive：** 4-8TB
- **配合媒体服务器：** 8TB+（搭配 [Jellyfin](/zh-CN/blog/jellyfin-guide)）

### 能和 Tailscale 一起用吗？

可以。在服务器上部署 [Tailscale + Headscale](/zh-CN/blog/tailscale-headscale-guide)，就能从任何地方安全访问 Nextcloud，无需暴露端口。这是推荐的远程访问方案。

### Nextcloud 怎么处理文件版本？

Nextcloud 默认保留无限文件版本。在 `config.php` 中配置：

```php
'versions_retention_obligation' => 'auto, 60',
'max_versions' => 100,
```

这意味着保留 60 天内或最多 100 个版本，以先到者为准。
