## 信息过载时代，你需要一个 AI 帮你筛新闻

每天打开 Hacker News、Reddit、Twitter，上百条内容刷过去，真正有价值的可能不到 5 条。手动筛选的时间成本越来越高，而错过重要信息的焦虑感却越来越强。

Horizon 是一个开源项目（GitHub 8.2k Stars），用 AI 自动完成信息源聚合、评分、去重、总结的全流程，最终生成一份结构化的中英双语日报。它不是又一个 RSS 阅读器，而是一个**信息过滤系统**——你只看 AI 认为值得你看的内容。

## Horizon 是什么

一句话定位：**AI 驱动的个人新闻雷达，自部署日报生成系统**。

核心流程：

```
信息源 → AI 评分（0-10）→ 去重合并 → 内容丰富 → 生成日报 → 多渠道分发
```

与传统 RSS 阅读器的区别在于：你不需要逐条浏览，AI 会帮你筛选并排序，只把高分内容推送到你面前。

## 功能全景

### 支持 7 种信息源

| 信息源 | 说明 |
|--------|------|
| Hacker News | 技术社区头条，自动抓取热门文章和评论 |
| RSS / Atom | 任意 RSS 源，兼容主流订阅格式 |
| Reddit | 按 subreddit 聚合，支持多社区 |
| Telegram | 频道消息监控 |
| Twitter/X | 推文抓取和分析 |
| GitHub | Trending 仓库和开发者动态 |
| OpenBB | 金融新闻和市场动态 |

### AI 能力

- **评分系统**：使用 Claude / GPT / Gemini / DeepSeek / Ollama 等模型，为每条内容打 0-10 分
- **新闻去重**：跨平台识别相同故事，合并为一条
- **内容丰富**：为陌生概念自动补充背景知识
- **社区讨论**：收集 HN / Reddit 等平台的评论摘要

### 输出方式

- GitHub Pages 静态站点
- 邮件订阅
- 飞书 / 钉钉 / Slack / Discord Webhook
- MCP Server（可接入其他 AI 工具）

## 快速开始

### 方式一：Docker Compose 部署（推荐）

创建 `docker-compose.yml`：

```yaml
services:
  horizon:
    build: .
    container_name: horizon
    volumes:
      - ./data:/app/data
    environment:
      - TZ=Asia/Shanghai
    restart: unless-stopped
```

配置文件 `data/config.json`（最小配置）：

```json
{
  "sources": [
    { "type": "hackernews", "enabled": true },
    { "type": "rss", "enabled": true, "feeds": ["https://example.com/feed.xml"] }
  ],
  "ai": {
    "provider": "openai",
    "api_key": "${AI_API_KEY}",
    "model": "gpt-4o-mini"
  },
  "output": [
    { "type": "github_pages", "enabled": true }
  ]
}
```

启动：

```bash
# 创建目录
mkdir -p data

# 设置 API Key
export AI_API_KEY="sk-your-key-here"

# 启动
docker compose run --rm horizon

# 查看日志
docker compose logs -f
```

### 方式二：本地运行

```bash
# 克隆仓库
git clone https://github.com/Thysrael/Horizon.git
cd Horizon

# 安装依赖（推荐使用 uv）
uv sync

# 配置
cp config.example.json data/config.json
# 编辑 config.json

# 运行
uv run horizon
```

## 进阶配置

### 配置向导

Horizon 内置配置向导，根据你的兴趣自动生成配置：

```bash
uv run horizon-wizard
```

向导会询问你感兴趣的话题领域（如 AI、Web 开发、DevOps），然后自动推荐相关的信息源和关键词。

### 分类过滤

在 `config.json` 中定义分类规则：

```json
{
  "categories": [
    {
      "name": "AI/ML",
      "keywords": ["llm", "gpt", "claude", "machine learning", "neural network"],
      "min_score": 7
    },
    {
      "name": "Web 开发",
      "keywords": ["react", "vue", "typescript", "next.js"],
      "min_score": 6
    },
    {
      "name": "DevOps",
      "keywords": ["docker", "kubernetes", "terraform", "ci/cd"],
      "min_score": 5
    }
  ]
}
```

### 自定义模型

支持所有 OpenAI 兼容 API，可以接入各种 LLM：

```json
// Ollama 本地模型
{
  "ai": {
    "provider": "ollama",
    "base_url": "http://localhost:11434",
    "model": "llama3.1"
  }
}
```

```json
// DeepSeek API
{
  "ai": {
    "provider": "openai",
    "base_url": "https://api.deepseek.com",
    "api_key": "${DEEPSEEK_API_KEY}",
    "model": "deepseek-chat"
  }
}
```

```json
// Anthropic Claude
{
  "ai": {
    "provider": "anthropic",
    "api_key": "${ANTHROPIC_API_KEY}",
    "model": "claude-sonnet-4-20250514"
  }
}
```

## 自动化部署

### GitHub Actions 定时任务

在仓库根目录创建 `.github/workflows/daily.yml`：

```yaml
name: Daily News

on:
  schedule:
    - cron: "0 8 * * *"  # 每天 UTC 8:00（北京时间 16:00）
  workflow_dispatch:

jobs:
  generate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Run Horizon
        run: |
          docker compose run --rm horizon

      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v4
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./data/summaries
```

### Crontab 方式

如果使用自己的服务器，直接用 crontab：

```bash
# 每天下午 4 点运行
0 16 * * * cd /opt/horizon && docker compose run --rm horizon
```

## 与其他自部署工具的组合

Horizon 可以与你已有的自部署生态协同工作：

### 与 Ollama 配合

用本地 Ollama 运行 AI 评分，完全离线，无 API 费用：

```json
{
  "ai": {
    "provider": "ollama",
    "base_url": "http://ollama.local:11434",
    "model": "llama3.1:70b"
  }
}
```

如果你还没搭建 Ollama，参考博客中的 [Ollama + Open WebUI 本地 AI 全家桶](/zh-cn/blog/ollama-openwebui-guide)。

### 与 n8n 配合

用 n8n 编排更复杂的信息流：Horizon 生成日报 → n8n 解析 → 自动发送到飞书群 / Notion / 数据库。

参考 [n8n 完全指南](/zh-cn/blog/n8n-guide) 了解工作流编排细节。

### 与 Uptime Kuma 配合

监控 Horizon 的运行状态，确保日报每天准时生成：

```yaml
# Uptime Kuma 监控配置
- type: HTTP(S)
  url: http://horizon.local:8080/health  # 具体端点请参考 Horizon 文档
  interval: 300
```

参考 [Uptime Kuma 自部署指南](/zh-cn/blog/uptime-kuma-guide) 搭建监控。

## 总结

Horizon 解决的是信息过载问题：用 AI 替代人工筛选，让每天的信息获取从"刷 100 条"变成"读 10 条精选"。

### 快速行动清单

1. **选择部署方式**：Docker Compose（推荐）或本地运行
2. **配置信息源**：至少添加 Hacker News + 1 个 RSS 源
3. **设置 AI 模型**：OpenAI / Claude / 本地 Ollama 任选
4. **配置输出渠道**：GitHub Pages 或邮件订阅
5. **设置定时任务**：GitHub Actions 或 crontab，每天自动生成

整个过程 10-15 分钟，之后每天打开邮箱或网页就能看到 AI 帮你筛选的新闻日报。
