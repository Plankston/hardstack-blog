## Information Overload? Let AI Filter Your News

Every day you open Hacker News, Reddit, Twitter — hundreds of posts fly by, and maybe 5 are actually worth reading. The time cost of manual filtering keeps rising, while the anxiety of missing important information grows stronger.

Horizon (8.2k GitHub Stars) uses AI to automate the entire pipeline: source aggregation, scoring, deduplication, and summarization — generating a structured bilingual daily briefing. It's not another RSS reader; it's an **information filtering system** where you only see what AI thinks is worth your time.

## What Is Horizon

One-liner: **AI-powered personal news radar with self-hosted daily briefing generation.**

Core flow:

```
Sources → AI Scoring (0-10) → Dedup & Merge → Content Enrichment → Daily Briefing → Multi-channel Distribution
```

The difference from traditional RSS readers: instead of browsing every item, AI ranks and sorts them, pushing only high-score content to you.

## Feature Overview

### 7 Supported Sources

| Source | What It Does |
|--------|-------------|
| Hacker News | Tech community headlines, auto-fetch top stories and comments |
| RSS / Atom | Any RSS feed, compatible with all standard formats |
| Reddit | Subreddit posts + user activity |
| Telegram | Public channel monitoring |
| Twitter/X | Tweet fetching and analysis |
| GitHub | Trending repos and developer activity |
| OpenBB | Financial news and market dynamics |

### AI Capabilities

- **Scoring System**: Uses Claude / GPT / Gemini / DeepSeek / Ollama to score each item 0-10
- **News Deduplication**: Cross-platform detection of duplicate stories, merged into one
- **Content Enrichment**: Auto-supplements background knowledge for unfamiliar concepts
- **Community Discussion**: Collects and summarizes comments from HN, Reddit, etc.

### Output Channels

- GitHub Pages static site
- Email subscription
- Feishu / DingTalk / Slack / Discord Webhook
- MCP Server (integrate with other AI tools)

## Quick Start

### Option 1: Docker Compose (Recommended)

Create `docker-compose.yml`:

```yaml
services:
  horizon:
    build: .
    container_name: horizon
    volumes:
      - ./data:/app/data
    environment:
      - TZ=Asia/Shanghai
```

Create `data/config.json` (minimal config):

```json
{
  "ai": {
    "provider": "openai",
    "model": "gpt-4o-mini",
    "api_key_env": "OPENAI_API_KEY"
  },
  "sources": {
    "rss": [
      {
        "name": "Simon Willison",
        "url": "https://simonwillison.net/atom/everything/"
      }
    ]
  },
  "filtering": {
    "ai_score_threshold": 6.0
  }
}
```

Run:

```bash
# Set API Key
export OPENAI_API_KEY="sk-your-key-here"

# Start
docker compose run --rm horizon

# Verify daily briefing generated
ls data/summaries/
```

### Option 2: Local Installation

```bash
git clone https://github.com/Thysrael/Horizon.git
cd Horizon

# Install dependencies (uv recommended)
uv sync

# Configure
cp data/config.example.json data/config.json

# Run
uv run horizon
```

## Advanced Configuration

### Setup Wizard

Horizon has a built-in wizard that generates config based on your interests:

```bash
uv run horizon-wizard
```

The wizard asks about your interest areas (AI, Web Dev, DevOps, etc.) and recommends relevant sources.

### Category Filtering

Define category rules in `config.json`:

```json
{
  "filtering": {
    "categories": [
      {
        "name": "AI/ML",
        "keywords": ["llm", "gpt", "claude", "machine learning"],
        "min_score": 7
      },
      {
        "name": "Web Dev",
        "keywords": ["react", "vue", "typescript", "next.js"],
        "min_score": 6
      }
    ]
  }
}
```

### Custom AI Models

Supports all OpenAI-compatible APIs — connect any LLM:

```json
// Ollama (local, free)
{
  "ai": {
    "provider": "ollama",
    "base_url": "http://localhost:11434",
    "model": "llama3.1"
  }
}

// DeepSeek
{
  "ai": {
    "provider": "openai",
    "base_url": "https://api.deepseek.com",
    "api_key_env": "DEEPSEEK_API_KEY",
    "model": "deepseek-chat"
  }
}

// Anthropic Claude
{
  "ai": {
    "provider": "anthropic",
    "api_key_env": "ANTHROPIC_API_KEY",
    "model": "claude-sonnet-4-20250514"
  }
}
```

## Automation

### GitHub Actions Cron Job

Create `.github/workflows/daily.yml`:

```yaml
name: Daily News

on:
  schedule:
    - cron: "0 8 * * *"  # UTC 8:00 (Beijing 16:00)
  workflow_dispatch:

jobs:
  generate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Run Horizon
        run: docker compose run --rm horizon

      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v4
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./data/summaries
```

### Crontab

```bash
# Run daily at 4 PM
0 16 * * * cd /opt/horizon && docker compose run --rm horizon
```
