## Why Not Otter.ai / Fireflies.ai

| 对比项 | Otter.ai | Fireflies.ai | Meetily |
|--------|----------|--------------|---------|
| **数据位置** | Otter 服务器 | Fireflies 服务器 | 你自己的电脑 |

**真正的问题不是价格。** Otter.ai 的免费版每月 300 分钟转录，对轻度用户够用。问题是**控制权**。你的会议录音包含客户信息、商业策略、产品路线图——这些数据在别人的服务器上，你无法审计、无法删除、无法保证它不被用于其他用途。

**The real issue isn't price.** Otter.ai's free tier gives 300 minutes per month — enough for light users. The issue is **control**. Your meeting recordings contain client information, business strategy, product roadmaps — data on someone else's servers that you can't audit, can't delete, can't guarantee won't be used for other purposes.

我之前用 Otter.ai 两年，累积了 400+ 小时的会议记录。有一天我导出数据，发现它只支持 CSV 格式——没有原始音频，没有时间戳对齐，连说话人都没区分。我花了三天手动整理。

I used Otter.ai for two years, accumulating 400+ hours of meeting records. One day I exported my data and found it only supports CSV — no raw audio, no timestamp alignment, no speaker diarization. It took me three days to manually organize.

**Meetily 的态度很明确：你的数据是你的，永远。**

## What It Can Do

Meetily 不只是转录。它是一个完整的会议工作流：

Meetily isn't just transcription. It's a complete meeting workflow:

1. **实时转录** — 会议进行中，文字同步显示。用 Whisper 或 Parakeet 模型，比 Otter.ai 快 4 倍。
2. **说话人分离** — 自动区分谁在说话，不用手动标注。
3. **AI 摘要** — 会后自动生成结构化摘要：要点、待办事项、关键决策。
4. **本地存储** — 所有数据存 SQLite，你随时可以查询、导出、删除。
5. **多平台支持** — Zoom、Teams、Google Meet、Webex，通吃。

1. **Real-time transcription** — Text appears as the meeting happens. Powered by Whisper or Parakeet, 4x faster than Otter.ai.

3. **AI summaries** — Auto-generates structured summaries post-meeting: key points, action items, decisions.

### 方案三：Linux 从源码编译

### Option 3: Build from Source on Linux

Linux 目前没有预编译的桌面应用，需要从源码编译：

Linux doesn't have a pre-built desktop app yet. Build from source:

```bash
# 安装依赖
# Install dependencies
sudo apt update
sudo apt install -y curl wget git build-essential cmake \
  libwebkit2gtk-4.1-dev libappindicator3-dev librsvg2-dev \
  patchelf libssl-dev libgtk-3-dev libayatana-appindicator3-dev

# 克隆仓库
# Clone the repository
git clone https://github.com/Zackriya-Solutions/meetily.git
cd meetily

# 安装 Rust（如果还没装）
# Install Rust (if not already installed)
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source ~/.cargo/env

# 安装前端依赖
# Install frontend dependencies
cd frontend
npm install
cd ..

# 安装后端依赖
# Install backend dependencies
cd backend
pip install -r requirements.txt
cd ..

# 构建
# Build
cargo tauri build
```

编译完成后，可执行文件在 `src-tauri/target/release/` 目录下。

After compilation, the executable is in `src-tauri/target/release/`.

## 首次配置

## First-Time Configuration

### 选择转录模型

### Choose a Transcription Model

Meetily 支持两种转录引擎：

Meetily supports two transcription engines:

| 模型 | 速度 | 准确度 | 内存需求 | 推荐场景 |
|------|------|--------|----------|----------|
| Whisper base | 快 | 中等 | 1GB | 日常会议，英语为主 |
| Whisper large-v3 | 慢 | 高 | 4GB | 重要会议，多语言 |
| Parakeet | 最快 | 高 | 2GB | 实时转录，性能优先 |

| Model | Speed | Accuracy | Memory | Best For |
|-------|-------|----------|--------|----------|
| Whisper base | Fast | Medium | 1GB | Daily meetings, English-focused |
| Whisper large-v3 | Slow | High | 4GB | Important meetings, multilingual |
| Parakeet | Fastest | High | 2GB | Real-time transcription, performance-first |

**我的选择：** 日常用 Whisper base，重要会议切换到 large-v3。Parakeet 是新出的，速度快但生态还在完善。

**My pick:** Whisper base for daily use, switch to large-v3 for important meetings. Parakeet is newer and fast, but its ecosystem is still maturing.

**推荐用 Ollama。** 如果你还没装 Ollama，跟着[前面的教程](/zh-CN/blog/ollama-openwebui-guide)装一下。Meetily 会自动检测本地的 Ollama 实例。

## 常见问题和解决方案

## Common Issues and Fixes

### 问题 1：转录很慢

### Issue 1: Transcription Is Slow

**原因：** 你在用 CPU 跑 Whisper large-v3。

**Cause:** You're running Whisper large-v3 on CPU.

**解决：**
1. 切换到 Whisper base 模型——速度快 3 倍，准确度损失很小。
2. 如果有 NVIDIA GPU，确保 CUDA 驱动正确安装。
3. 用 Parakeet 模型——专为速度优化。

**Fix:**
1. Switch to Whisper base — 3x faster with minimal accuracy loss.
2. If you have an NVIDIA GPU, ensure CUDA drivers are installed correctly.
3. Use Parakeet — optimized for speed.

### 问题 2：中文转录不准确

### Issue 2: Chinese Transcription Is Inaccurate

**原因：** 默认的 Whisper base 模型对中文支持一般。

**Cause:** Default Whisper base model has mediocre Chinese support.

**解决：** 切换到 Whisper large-v3 或 medium 模型。large-v3 的中文准确率明显更高。

**Fix:** Switch to Whisper large-v3 or medium model. large-v3 has significantly better Chinese accuracy.

### 问题 3：Ollama 摘要生成失败

### Issue 3: Ollama Summary Generation Fails

**原因：** Ollama 没运行，或者模型没下载。

**Cause:** Ollama isn't running, or the model isn't downloaded.

**解决：**

**Fix:**

```bash
# 检查 Ollama 是否运行
# Check if Ollama is running
curl http://localhost:11434/api/tags

# 如果没运行，启动它
# If not running, start it
ollama serve

# 确保模型已下载
# Ensure model is downloaded
ollama pull llama3.1:8b
```

### 问题 4：会议软件没声音

### Issue 4: No Audio from Meeting Software

**原因：** Meetily 没有捕获到正确的音频流。

**Cause:** Meetily isn't capturing the correct audio stream.

**解决：**
1. 检查系统音频设置——Meetily 需要捕获"系统音频"，不只是麦克风。
2. macOS 用户：需要在系统偏好设置中给 Meetily 授权"录音"权限。
3. Windows 用户：确保 Meetily 以管理员身份运行。

**Fix:**
1. Check system audio settings — Meetily needs to capture "system audio", not just microphone.
2. macOS users: Grant Meetily "recording" permission in System Preferences.
3. Windows users: Ensure Meetily runs as administrator.

## Honest Limitations

Meetily 不是完美的。说几个实际遇到的问题：

Meetily isn't perfect. Here are real issues I've encountered:

4. **模型文件较大** — Whisper large-v3 约 3GB，下载需要时间。
5. **社区相对年轻** — 不像 Otter.ai 那样有十年积累，遇到问题可能需要自己翻 GitHub Issues。

**但这些局限性值不值得接受？** 我的答案是：绝对值得。因为核心价值——**数据控制权**——是任何 SaaS 工具给不了你的。

**But are these limitations worth accepting?** My answer: absolutely. Because the core value — **data control** — is something no SaaS tool can give you.
