## 0. 一句话项目定义

建立一个**英文为主、中文为辅**的技术博客，专注**开源/自托管/AI 工具类项目的深度安装教程与踩坑记录**，通过 SEO 流量、联盟营销和广告位实现长期被动收入。

---

## 1. 为什么是 2026 年最好的个人内容创业方向之一

### 1.1 三个不会变的趋势

1. **AI 工具爆发的内容真空**：每周有 500+ 新 GitHub 项目上线，但只有 < 5% 有完整中文/英文安装教程
2. **自托管复兴**：Notion / Slack 涨价 + 隐私意识 → 自托管需求持续增长
3. **AI 降低生产门槛**：原来 1 篇文章 15 小时 → 现在 5-6 小时，且质量更高

### 1.2 为什么不做"短视频"或"AI 工具聚合站"

| 方向 | 优点 | 致命问题 |
|------|------|---------|
| 短视频（抖音/B站）| 起量快 | 平台依赖、内容沉没、不积累 |
| 工具聚合站（Product Hunt 镜像）| 流量大 | 同质化、无用户黏性、SEO 难 |
| 这次的方向（深度教程站）| 资产积累、长尾 SEO、被动收入 | 起量慢、需坚持 6+ 月 |

**本质区别：**
- 短视频 = 流量在平台
- 这次 = 流量在自己网站

## 2. 定位与差异化

### 2.1 目标用户画像

**主用户（英文版 70% 流量来源）：**
- 自托管爱好者（Reddit r/selfhosted 130 万订阅）
- 独立开发者 / 创业者
- 系统管理员 / DevOps
- 隐私敏感型技术用户

**次用户（中文版 30% 流量来源）：**
- 国内程序员（HelloGitHub 读者）
- AI 工具早期采用者
- 准备出海的技术团队

### 2.2 与对标账号的差异化

| 对标 | 形态 | 我们的差异 |
|------|------|-----------|
| **HelloGitHub** | 一句话推荐清单 | 我们做完整 5000-10000 字深度教程 |
| **Pieter Levels / nomadlist** | 偏独立开发产品 | 我们做工具和开源项目 |
| **自建站教程类（wselfhost.com 等）** | 偏文字 | 我们加视频、加踩坑、加横向对比 |
| **YouTube 自托管频道** | 视频为主 | 我们做图文（更适合 SEO） |

### 2.3 核心内容承诺（用户选择我们的原因）

1. **我亲自装过**（不是云测评）
2. **真实报错截图**（不是完美教程）
3. **3 个月后回访更新**（不是发完就忘）
4. **有对比有推荐**（"X vs Y 我推荐 X，因为..."）
5. **可被搜索引擎搜到**（SEO 友好，不是藏在社交媒体）

## 4. 技术栈

### 4.1 推荐方案（性价比 + 易维护）

| 组件 | 选择 | 理由 | 成本 |
|------|------|------|------|
| 博客框架 | **Astro** | 现代、内容站首选、MDX、内置 SEO | 免费 |
| 样式 | Tailwind CSS | 主流、易定制 | 免费 |
| 内容管理 | MDX 文件 + Git | 纯文本，不依赖 CMS | 免费 |

| 评论 | **Giscus**（GitHub Discussions）| 零成本、技术用户友好 | 免费 |
| 搜索 | **Pagefind** | 静态搜索，零成本 | 免费 |
| 邮件订阅 | **Buttondown** 或 Substack | 个人起步免费，自动化 | $0-9/月 |
| 监控 | Plausible Analytics | 隐私友好、轻量 | $9/月（可选）|
| 截图 | ShareX + PicGo | 自动上传到图床 | 免费 |
| 图床 | Cloudflare R2 | 便宜、10GB 免费 | $0-1/月 |

**总固定成本：$10-30/月 = ¥80-200/月**

### 5.1 五大内容类型

| 类型 | 占比 | 例子 |
|------|------|------|
| **安装教程** | 50% | "Immich 完整安装指南" |
| **横向对比** | 20% | "Vaultwarden vs Bitwarden vs 1Password" |
| **踩坑合集** | 15% | "我装 Open WebUI 时遇到的 7 个坑" |
| **新闻解读** | 10% | "Awesome-GPTs 项目是怎么做起来的" |
| **盘点榜单** | 5% | "2026 年最值得自托管的 10 个项目" |

**关键洞察：**
- "X vs Y" 类内容是**搜索量最大**的（"best X alternative" "X vs Y"）
- "complete guide" 类内容是**外链最多**的
- 踩坑类内容是**用户最信任**的

### 5.2 选题来源（每天 30 分钟）

**主源（70%）：**
- GitHub Trending（每日榜）
- Product Hunt（每日榜）
- Reddit r/selfhosted（热度帖）
- Hacker News（首页 > 100 赞）
- HelloGitHub 月榜

**辅源（30%）：**
- V2EX、掘金、知乎热榜
- 微博科技话题
- Twitter/X 关注 KOL
- YouTube 趋势（看哪些视频被翻成中文）

### 5.3 选题评估表（每个选题打分，决定是否做）

## 6. AI 工作流（核心效率系统）

### 6.1 单篇内容生产时间分配

```
选题评估         30min   （人工）
自己实装         2-3h    （人工，必做）
写技术笔记       1h      （人工）
AI 改写为教程   30min   （AI）
配图截图         30min   （人工）
AI 翻译为英文   30min   （AI）
SEO 优化         20min   （AI）
发布与外链      30min   （人工）
────────────────────────────────
总计            6-8h     前期
                 4-5h     熟练后
```

### 6.2 核心 AI 提示词库

#### 提示词 1：笔记 → 教程改写
```
你是技术写作专家。基于我的实操笔记，写一篇深度安装教程。

要求：
1. 结构清晰：概述、前置要求、安装步骤（带命令）、配置、踩坑、FAQ
2. 命令完整可复制，不要省略
3. 加入 5 个真实可能的报错及解决方案
4. 适合中级用户，不要太基础
5. 字数 5000-8000 字

我的笔记：
[粘贴你的笔记]
```

#### 提示词 2：英文润色
```
你是 native English 技术作家。把以下中文翻译为地道英文教程。

要求：
1. 流畅自然，不带中式英语
2. 技术术语使用行业标准翻译
3. 保留所有代码块不变
4. 风格：直接、实用、不啰嗦
5. 适合 Hacker News 读者群

原文：
[粘贴中文]
```

#### 提示词 3：SEO 优化
```
基于以下教程，生成 SEO 友好的：
1. 标题（包含主关键词，< 60 字符）
2. Meta description（< 160 字符，含 CTA）
3. URL slug（短、含关键词）
4. 5 个相关长尾关键词
5. 内链建议（链到博客内其他教程）

教程内容：
[粘贴正文]
```

#### 提示词 4：踩坑挖掘
```
基于以下教程内容，列出 7 个最可能踩的坑，每个包括：
- 坑的描述（具体场景）
- 报错信息（真实可能）
- 解决步骤（具体命令或操作）
- 预防建议

教程内容：
[粘贴正文]
```

## 10. 风险与避坑清单

### 10.1 内容侧风险
- ❌ **追新项目**：选 6-12 个月后还在被讨论的，新项目流量未验证
- ❌ **堆量不堆质**：30 篇精品 > 200 篇水稿
- ❌ **不维护**：3 个月回访更新是承诺，必须做到
- ❌ **AI 痕迹重**：AI 初稿必须人工改 30%+

### 10.2 SEO 风险
- ❌ **不做关键词研究**：闭门造车，无人搜的内容
- ❌ **不做外链**：好内容也需主动外链
- ❌ **不做内链结构**：站内孤岛
- ❌ **关键词堆砌**：被 Google 惩罚

### 10.3 平台依赖风险
- ❌ **完全靠 Google**：备用 Bing、Yandex
- ❌ **完全靠联盟**：联盟政策可能变（多平台分散）

### 10.4 心理/时间风险
- ❌ **3 个月没收入就放弃**：这行的真实节奏是 6-12 月才见效
- ❌ **每周发 5 篇**：质量下降，疲劳感
- ❌ **不回复评论**：社区感丧失
- ❌ **独自闷头做**：需要找 1-2 个 peer 互相打气
